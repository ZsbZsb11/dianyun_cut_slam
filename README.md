# dianyun_cut_slam

ROS 2 Humble + Gazebo Fortress indoor robot simulation for RGB-D based 2D SLAM and Nav2 navigation.

The project builds a two-room home environment and a differential-drive robot with IMU and RGB-D camera. There is no physical lidar in the model. A 2D `/scan` topic is generated by slicing the RGB-D point cloud with `pointcloud_to_laserscan`, then used by `slam_toolbox` and Nav2.

## Features

- Two-room Gazebo home world.
- Xacro/URDF and SDF robot description.
- RGB-D camera, IMU, odometry and `/cmd_vel` bridge.
- RGB-D point cloud to 2D laser scan pipeline.
- Online mapping with `slam_toolbox`.
- Nav2 navigation with included map files and RViz configs.
- Helper scripts for cloud, scan and RGB-D pipeline inspection.

## Repository Structure

```text
src/
  home_sim_description/   Robot model, URDF/Xacro and Gazebo model files
  home_sim_gazebo/        Gazebo world, bridge and simulation launch files
  home_sim_navigation/    SLAM, Nav2, maps, RViz and RGB-D scan launch files
tools/                    Debug and verification scripts
```

## Environment

Recommended environment:

- Ubuntu 22.04
- ROS 2 Humble
- Gazebo Fortress
- `colcon`

Install the main ROS dependencies:

```bash
sudo apt update
sudo apt install \
  ros-humble-ros-gz-sim \
  ros-humble-ros-gz-bridge \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox \
  ros-humble-pointcloud-to-laserscan \
  ros-humble-xacro \
  ros-humble-robot-state-publisher \
  ros-humble-teleop-twist-keyboard
```

If your ROS Humble installation uses older Gazebo package names, replace `ros-humble-ros-gz-*` with the matching `ros-humble-ros-ign-*` packages.

## Build

```bash
cd ~/home_sim_project
source /opt/ros/humble/setup.bash
colcon build --symlink-install
source install/setup.bash
```

Keep the repository `src/` layout unchanged when copying the project into WSL2 or a Linux ROS workspace.

## Run Simulation

Start Gazebo server and robot simulation:

```bash
ros2 launch home_sim_gazebo sim.launch.py
```

Start with the Gazebo GUI:

```bash
ros2 launch home_sim_gazebo sim.launch.py gui:=true
```

Expected topics include:

- `/cmd_vel`
- `/odom`
- `/imu/data`
- `/camera/color/image_raw`
- `/camera/depth/image_raw`
- `/camera/points`

## Mapping

Launch the simulation, RGB-D scan conversion and `slam_toolbox`:

```bash
ros2 launch home_sim_navigation mapping.launch.py
```

Drive the robot in another terminal:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

Save a generated map:

```bash
ros2 run nav2_map_server map_saver_cli -f ~/home_sim_project/src/home_sim_navigation/maps/generated_home_map
```

## Navigation

Launch Gazebo, RGB-D scan conversion and Nav2:

```bash
ros2 launch home_sim_navigation navigation.launch.py
```

Use RViz2 to set the initial pose and send navigation goals between rooms.

The navigation launch currently uses:

- `src/home_sim_navigation/config/my_nav2_params.yaml`
- `src/home_sim_navigation/maps/rgbd_two_room_map_dense_views_cleaned.yaml`

## Debugging

Useful RViz/debug launch files:

```bash
ros2 launch home_sim_navigation rviz_mapping.launch.py
ros2 launch home_sim_navigation rviz_odom.launch.py
ros2 launch home_sim_navigation rviz_debug.launch.py
```

Helper tools:

```bash
python3 tools/inspect_cloud.py
python3 tools/inspect_scan.py
bash tools/verify_rgbd_pipeline.sh
```

## Notes

- Lidar is intentionally excluded; do not use lidar-specific launch/config files.
- The RGB-D slice settings are in `src/home_sim_navigation/config/pointcloud_to_laserscan.yaml`.
- If `/scan` is sparse or empty, adjust `min_height` and `max_height`. The current slice is around `0.18` to `0.22` meters.
- Gazebo topic names may vary across Fortress bridge versions. If camera topics do not appear, run `gz topic -l` or `ign topic -l` and update the bridge arguments in `src/home_sim_gazebo/launch/sim.launch.py`.
