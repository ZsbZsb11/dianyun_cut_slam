# ROS2 Humble + Gazebo Fortress Home Simulation

This workspace builds a two-room home simulation for RGB-D based 2D SLAM and
Nav2 navigation. The robot has a differential-drive base, IMU, and RGB-D
camera. It does not include a lidar. The `/scan` topic is generated by slicing
the RGB-D point cloud at 18-22 cm height with `pointcloud_to_laserscan`.

## Packages

- `home_sim_description`: robot URDF/Xacro and Gazebo SDF model.
- `home_sim_gazebo`: Gazebo Fortress two-room home world and simulation launch.
- `home_sim_navigation`: RGB-D point cloud slicing, `slam_toolbox`, Nav2 params,
  and maps copied from `https://github.com/Mao-uc/BPi/tree/main/params%26maps`.

## Requirements

Use Ubuntu 22.04 with ROS2 Humble and Gazebo Fortress. On a fresh system,
install the main dependencies:

```bash
sudo apt update
sudo apt install \
  ros-humble-ros-gz-sim \
  ros-humble-ros-gz-bridge \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox \
  ros-humble-pointcloud-to-laserscan \
  ros-humble-xacro \
  ros-humble-robot-state-publisher \
  ros-humble-teleop-twist-keyboard
```

If your Humble installation uses the older package names, replace
`ros-humble-ros-gz-*` with the matching `ros-humble-ros-ign-*` packages.

## Build

```bash
cd ~/home_sim_project
source /opt/ros/humble/setup.bash
colcon build --symlink-install
source install/setup.bash
```

If you copy this Windows desktop folder into WSL2, keep the `src/` directory
structure unchanged.

## Run Gazebo Only

```bash
ros2 launch home_sim_gazebo sim.launch.py
```

Expected ROS topics include:

- `/cmd_vel`
- `/odom`
- `/imu/data`
- `/camera/color/image_raw`
- `/camera/depth/image_raw`
- `/camera/points`

## Mapping

```bash
ros2 launch home_sim_navigation mapping.launch.py
```

In another terminal, drive the robot:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

Save the generated map:

```bash
ros2 run nav2_map_server map_saver_cli -f ~/home_sim_project/src/home_sim_navigation/maps/generated_home_map
```

## Navigation

The navigation launch uses the copied `my_nav2_params.yaml` and
`maps/my_map_scn.yaml` by default:

```bash
ros2 launch home_sim_navigation navigation.launch.py
```

Use RViz2 to set the initial pose and send Nav2 goals between the living room
and bedroom.

## Important Notes

- Do not launch `ydlidar_x3.yaml`; lidar is intentionally excluded.
- The initial point cloud slice is configured in
  `src/home_sim_navigation/config/pointcloud_to_laserscan.yaml`.
- If `/scan` is sparse or empty, adjust `min_height` and `max_height` around the
  robot frame. The current default is `0.18` to `0.22` meters.
- Gazebo topic names can vary slightly across Fortress bridge packages. If the
  camera topics do not appear, run `gz topic -l` or `ign topic -l` and update
  `src/home_sim_gazebo/launch/sim.launch.py` bridge arguments accordingly.
